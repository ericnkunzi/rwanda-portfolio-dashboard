Current Data Early-Warning — Based Only on Existing Kobo Fields
This panel classifies each record using simple management logic from existing Kobo fields only: Immediate attention means delayed + reported problem; Early warning means a reported problem before/without overdue status; and Overdue without problem means a delayed record requiring verification of the reason. It does not estimate financial progress, physical progress, exchange-rate exposure, or closing-date risk when those fields are not captured.
Immediate attention—Delayed records with reported problems.
Early warning—Problems reported before/without overdue flag.
Sector concentration—Highest meaningful delay/problem concentration; small “Other” samples are not allowed to dominate.
Data confidence—Records assessed with current fields.

06Beneficiaries & Reach
Beneficiary methodology: The total below is calculated from the Portfolio Mapping records only, because the Activity Mapping file contains many GPS/site-level entries for the same project and would overstate beneficiaries if summed. For each unique project/programme, the dashboard counts the highest reported beneficiary value once, then sums those project-level values. This logic updates automatically when new live data is loaded.
Total beneficiaries—Project-level total: max reported beneficiaries per unique project/programme, counted once.
Projects counted—Unique project/programme groups with beneficiary values above zero.
Latest beneficiary data—Latest available data collection date among beneficiary records.
